# Уровень 1: Обзор проекта

## Назначение

**test_api_evo_link** — Telegram-бот для генерации изображений по шаблонным промптам. Пользователь входит по паролю, выбирает один из промптов (настраиваемых админом), подставляет значения в переменные шаблона, после чего бот отправляет запрос во внешний API (Evolink) и возвращает сгенерированное изображение. Учёт токенов (баланс) и промокоды поддерживаются.

## Основные возможности

- **Пользователи:** авторизация по паролю, выбор промпта, ввод переменных (текст/опции/своё значение), генерация изображения, списание токенов, активация промокодов по ссылке `/start <code>`.
- **Админы:** управление промптами (создание, редактирование шаблона и переменных, референсное фото, примеры), управление промокодами, выдача баланса (`/addme`), настройка генерации (модель, размер, качество), опциональная интеграция с DeepSeek для «feach» (идеи и фичи по промпту).

## Внешние зависимости

| Зависимость        | Назначение |
|--------------------|------------|
| **Telegram (Bot API)** | Приём сообщений и callback'ов, отправка текста и изображений. |
| **PostgreSQL**     | Пользователи, промпты, промокоды, баланс, состояние бота. |
| **Evolink API**    | Создание задачи генерации изображения, опрос статуса, получение результата. |
| **DeepSeek (опционально)** | Генерация идей и фич по промпту (feach) для удобного конфига переменных. |

## Потоки данных (высокоуровнево)

1. **Старт:** `/start` → проверка/создание пользователя в БД → при наличии payload — погашение промокода → если не авторизован — запрос пароля → иначе показ кнопок с промптами.
2. **Генерация:** выбор промпта → пошаговый ввод переменных (текст/кнопки/своё) → сбор референсных фото и URL → проверка баланса → создание задачи в Evolink → опрос до завершения → списание токена → отправка изображения пользователю.
3. **Админка:** создание/редактирование промптов (шаблон, переменные, референс, примеры), промокоды, настройки, экспорт/импорт промптов (JSON).

## Технологический стек

- **Язык:** Python 3
- **Фреймворк бота:** aiogram (FSM, Router, filters)
- **БД:** asyncpg (PostgreSQL)
- **HTTP-клиент:** aiohttp (Evolink API)
- **Конфиг:** dotenv, переменные окружения (`.env`)

## Точка входа

- `bot.py` — загрузка настроек, создание пула БД, инициализация `Repo`, создание `Bot` и `Dispatcher`, подключение роутера с хендлерами, запуск long polling.
